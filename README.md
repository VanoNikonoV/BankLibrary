# BankLibrary - Учебный проект по курсу C# Developer

Данный репозиторий создан в рамках прохождения курса **"C# – разработчик с нуля до PRO"** на платформе [Skillbox](https://go.skillbox.ru/).

Этот проект представляет собой эволюцию банковской системы, от простого оконного приложения до многопроектного решения с продвинутой логикой. Код наглядно демонстрирует поэтапное освоение ключевых концепций языка C# и платформы .NET.

### Основная цель проекта:
Продемонстрировать навыки разработки десктопных приложений с графическим интерфейсом (WPF) и применения фундаментальных принципов объектно-ориентированного программирования (ООП) для создания гибкой и расширяемой архитектуры.

---

### Стек технологий и ключевые концепции:
*   **Язык:** C#
*   **Платформа:** .NET (Core / 5+)
*   **Интерфейс:** Windows Presentation Foundation (WPF), XAML
*   **Архитектура:** Многослойная архитектура (разделение на проекты), Библиотеки классов
*   **ООП:** Инкапсуляция, Наследование, Полиморфизм
*   **Продвинутые механизмы:** Обобщения (Generics), Ковариантность и Контравариантность, Делегаты и События, Исключения, Методы расширения, Перегрузка операторов

---

### Этапы разработки (отражены в структуре коммитов и кода)

Проект развивался итеративно, что отражает реальный процесс обучения и усложнения требований.

**Модуль 1: Основы WPF**
*   Создание оконного интерфейса с использованием WPF.
*   Реализация простейших операций: разделение строки на слова и вывод в `ListBox`, перестановка слов в предложении и вывод в `Label`.
*   Демонстрирует навыки работы с базовыми элементами управления (TextBox, Button, ListBox, Label) и обработкой событий.

**Модуль 2: Принципы ООП (Инкапсуляция, Наследование, Полиморфизм)**
*   Создание классов для моделирования данных клиента и ролей сотрудников банка (Консультант, Менеджер).
*   **Инкапсуляция:** Реализация прав доступа. Консультант видит паспортные данные скрытыми (`*********`) и не может менять ФИО.
*   **Наследование:** Менеджер наследует функционал консультанта и расширяет его (возможность добавлять и редактировать все данные).
*   **Полиморфизм и Интерфейсы:** Введение интерфейсов для методов изменения данных, добавление логирования изменений (кто, когда и что изменил).

**Модуль 3: Обобщения (Generics) и Вариантность**
*   Реализация прототипа банковской системы с управлением счетами клиентов.
*   **Generics:** Создание параметризированных классов для работы со счетами, что обеспечивает типобезопасность.
*   **Ковариантность:** Реализация интерфейсов для пополнения счетов разных типов (депозитный/недепозитный).
*   **Контравариантность:** Создание механизма для переводов между счетами различных клиентов.

**Модуль 4: Делегаты и События**
*   Внедрение механизма оповещений и журналирования действий.
*   Использование **делегатов и событий** для уведомления об операциях (открытие/закрытие счета, перевод, пополнение, изменение данных клиента) с фиксацией времени и имени оператора.

**Модуль 5: Исключения, Методы Расширения и Многопроектность**
*   **Разделение на проекты:** Логика приложения вынесена в отдельную библиотеку классов (`BankLibrary`), а пользовательский интерфейс остался в проекте WPF. Это демонстрирует понимание принципов модульности и повторного использования кода.
*   **Исключения:** Создание и обработка пользовательских исключений (например, при некорректном вводе номера счета).
*   **Методы расширения и перегрузка операторов:** Добавлены для улучшения читаемости и функциональности кода (например, валидация данных или сравнение счетов).

### Как этот проект характеризует меня как разработчика:
*   **Обучаемость:** Способность последовательно усложнять проект, осваивая новые инструменты и парадигмы.
*   **Понимание основ:** Глубокое понимание принципов ООП, а не просто умение писать "рабочий" код.
*   **Внимание к архитектуре:** Умение разделять ответственность между классами и слоями приложения, стремление к созданию чистой и расширяемой архитектуры.
*   **Навыки работы с .NET:** Практический опыт работы с WPF, Generics, событиями и другими ключевыми механизмами платформы.
